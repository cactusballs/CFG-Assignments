.gitignore & requirements.txt

A .gitgnore file tells Git which files should be ignored/not pushed to the remote repository. They're normally reserved for temporary files, generated files and those that contain passwords. You can create one by following these steps:

  1. Open your terminal and navigate to the root directory of the project that needs the .gitignore file
  2. Code: 'touch .gitignore'
  3. If the command has succeeded there will be no output
  4. Open the .gitignore file and specify the files you want Git to ignore
  5. Commit and push as normal

A requirements.txt file tells a user which frameworks and libraries they'll need to use when working on a Python project.You can create one by following these steps:

  1. Open your terminal and navigate to the root directory of the Python project that needs a requirements.txt file
  2. Code: 'pip freeze > requirements.txt'. to generate a file that has the currently installed libraries and frameworks
  3. If needed you can edit the file to only include the necessary libraries and frameworks
  4. Commit and push as normal
